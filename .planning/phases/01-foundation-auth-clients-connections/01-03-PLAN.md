</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/clients/[id]/connections/* | Connect/reconnect initiation; carries active-client cookie |
| Next.js → External OAuth (Meta/LinkedIn) | Authorization-code + PKCE exchange |
| Next.js → PostgreSQL | Encrypted token ciphertext stored; plaintext never written |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Spoofing | OAuth `state` (CSRF + target confusion, D-06) | mitigate | `state` value stored in `oauthState` and verified at callback; active `clientId` bound in `oauthState` (not trusted from request body) — prevents target-confusion (PITFALL 6). |
| T-01-08 | Spoofing | PKCE verifier | mitigate | S256 `code_challenge`/`code_verifier` per RFC 7636; verifier checked at `exchangeCode`; stashed server-side in `oauthState`, never in the client. |
| T-01-09 | Info Disclosure | Token storage (CONN-03) | mitigate | AES-256-GCM at the boundary (`encrypt` before insert); plaintext never in DB/logs/API responses; key from `TOKEN_ENCRYPTION_KEY` env (CONN-03, PITFALL 14). |
| T-01-10 | Spoofing | `client_secret` exposure | mitigate | `META_CLIENT_SECRET` / `LINKEDIN_CLIENT_SECRET` server-only env; never shipped to browser bundle. |
| T-01-11 | Elevation | connect endpoints | mitigate | Every connect/reconnect route requires auth + `getActiveClientId()` scoping (reuses Plan 02 `requireClientScope`); `[id]` param validated against owner. |
| T-01-12 | Info Disclosure | LinkedIn no-refresh (PITFALL 1) | mitigate | `expiresAt` stored + surfaced via `statusFor`; "Reconnect required" + one-click re-auth (CONN-04); design does not assume refresh availability. |
| T-01-13 | Tampering | Meta short-lived token (PITFALL 2) | mitigate | `meta.exchangeCode` performs `fb_exchange_token` long-lived exchange before persistence; never stores the 1–2h token. |
</threat_model>
